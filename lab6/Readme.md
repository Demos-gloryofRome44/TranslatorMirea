# Лабораторная работа: транслятор на Flex + GNU Bison с семантикой

## Теоретическая справка

### Задачи компилятора

Перед генерацией кода исходный текст проходит три этапа.

1. **Лексический анализ (сканирование)** — поток символов разбивается на **лексемы (токены)**: ключевые слова, идентификаторы, литералы, операторы, разделители. Ошибки на этом этапе — **лексические** (незнакомый символ, нарушение правил записи константы и т.п.).

2. **Синтаксический анализ (разбор)** — последовательность токенов проверяется на соответствие **грамматике языка**. Если цепочка не выводима из аксиомы грамматики, фиксируется **синтаксическая ошибка**. На этом же этапе строится **абстрактное синтаксическое дерево (AST)**.

3. **Семантический анализ / исполнение** — по AST вычисляются значения выражений, выполняются операторы, поддерживается таблица переменных. Здесь фиксируются **семантические ошибки** (например, превышение лимита итераций цикла).

### Flex и Bison: разделение ролей

**Flex** (Fast Lexical Analyzer Generator) по описанию регулярных выражений и действий на совпадение генерирует программу на C — **лексер** `yylex()`, которая возвращает коды токенов. Типичный ввод: файл правил `.l`.

**GNU Bison** — генератор **LALR(1)**-парсеров (обобщённо: семейство LR; Bison строит таблицы сдвиг/свертка). По описанию грамматики в форме БНФ (файл `.y`) генерируется функция `yyparse()`, которая вызывает `yylex()` и проверяет синтаксис. При успехе разбор завершается **принятием** цепочки (accept), при несоответствии грамматике вызывается `yyerror()`. В действиях правил (`{ ... }`) строится AST через семантические значения (`%union`, `$$`, `$1`, …).

Связка **Flex + Bison** — стандартный способ получить разборщик на языке C: лексер «кормит» парсер целочисленными кодами токенов, объявлёнными директивой `%token` в файле Bison, вместе со значениями (имя идентификатора, числовое значение литерала).

### Контекстно-свободные грамматики

Язык описывается **КС-грамматикой**: правила имеют вид «один нетерминал → цепочка терминалов и нетерминалов». Конструкции вроде вложенного `while` и скобок в условии не задаются конечным автоматом без стека, поэтому для них нужен именно **контекстно-свободный** разбор (например, LR).

### Функциональность

Реализованы:

- **лексический анализатор** (Flex) — выдаёт токены и их значения;
- **синтаксический анализатор** (Bison) — проверяет грамматику и строит AST;
- **семантический анализатор (интерпретатор)** — обходит AST, исполняет программу, ведёт таблицу переменных, печатает результаты `print` и итоговые значения переменных.

### Грамматика

```text
<Program>     ::= <Statement>
<Statement>   ::= <While> | <Print> | <Expression> ";"
<While>       ::= "while" "(" <Condition> ")" <Statement>
<Condition>   ::= <Expression> <ComparisonOp> <Expression>
<Expression>  ::= <Primary> | "++" identifier | "--" identifier
<Primary>     ::= <identifier> | <number>
<ComparisonOp> ::= "<" | ">" | "<=" | ">=" | "==" | "!="
<Print>       ::= "print" "(" <Expression> ")" ";"
```

Идентификаторы и числа (целые и вещественные, в т.ч. с порядком `e`/`E`) задаются в лексере.

### Семантика

- Переменные хранятся в простой таблице `имя → значение` (double). Неинициализированная переменная при первом чтении создаётся со значением `0` (как в лабе 5).
- `++id` / `--id` — префиксный инкремент/декремент: `id ← id ± 1`, результат — новое значение.
- `<Expression> ";"` — выражение-оператор: вычисляется и результат отбрасывается (но побочные эффекты `++`/`--` применяются).
- `print(E);` — печать значения выражения (`E`) на отдельной строке. Целые печатаются без дробной части.
- `while (C) S` — цикл с предусловием. Для защиты от бесконечных циклов установлен лимит `MAX_ITERS = 10000` — при его превышении выдаётся **семантическая ошибка** и ненулевой код возврата.
- В конце печатается итоговое состояние таблицы переменных.

### Структура каталога

| Файл | Назначение |
|------|------------|
| `lexer.l` | Flex: токены, значения для `IDENTIFIER`/`NUMBER`, ошибка на недопустимом символе |
| `parser.y` | Bison: грамматика, `%union` с AST-узлами, `yyerror` с номером строки |
| `ast.h` / `ast.c` | Узлы AST и их конструкторы/освобождение |
| `semantic.h` / `semantic.c` | Интерпретатор AST: переменные, цикл, печать, лимит итераций |
| `main.c` | Точка входа: открывает файл из аргумента, вызывает `yyparse()` → `semantic_run()` |
| `Makefile` | Сборка: `bison -d -o y.tab.c`, `flex`, `gcc` |
| `tests/ok/` | Примеры корректных программ (успешный разбор и исполнение) |
| `tests/err/` | Примеры с ошибками (лексическими, синтаксическими, семантическими) |

### Сборка и запуск

Требования: **gcc**, **flex**, **GNU bison**.

```bash
cd lab6
make
```

Разбор и исполнение **только из файла** (путь передаётся первым аргументом):

```bash
./analyzer tests/ok/while_print.txt
```

Пример вывода:

```

=== Выполнение ===

2
3
4
5
6

=== Состояние переменных ===

a = 10
```

Автоматическая проверка набора тестов:

```bash
make test
```

Очистка артефактов сборки:

```bash
make clean
```
