## Теоретическая справка

## Задачи компилятора

Перед проверкой смысла программы (семантика) и генерацией кода исходный текст проходят два этапа.

1. **Лексический анализ (сканирование)** — поток символов разбивается на **лексемы (токены)**: ключевые слова, идентификаторы, литералы, операторы, разделители. Ошибки на этом этапе — **лексические** (незнакомый символ, нарушение правил записи константы и т.п.).

2. **Синтаксический анализ (разбор)** — последовательность токенов проверяется на соответствие **грамматике языка**. Если цепочка не выводима из аксиомы грамматики, фиксируется **синтаксическая ошибка**.

## Flex и Bison: разделение ролей

**Flex** (Fast Lexical Analyzer Generator) по описанию регулярных выражений и действий на совпадение генерирует программу на C — **лексер** `yylex()`, которая возвращает коды токенов. Типичный ввод: файл правил `.l`.

**GNU Bison** — генератор **LALR(1)**-парсеров (обобщённо: семейство LR; Bison строит таблицы сдвиг/свертка). По описанию грамматики в форме БНФ (файл `.y`) генерируется функция `yyparse()`, которая вызывает `yylex()` и проверяет синтаксис. При успехе разбор завершается **принятием** цепочки (accept), при несоответствии грамматике вызывается `yyerror()`.

Связка **Flex + Bison** — стандартный способ получить разборщик на языке C: лексер «кормит» парсер целочисленными кодами токенов, объявлёнными директивой `%token` в файле Bison.

## Контекстно-свободные грамматики

Язык, описывается **КС-грамматикой**: правила имеют вид «один нетерминал → цепочка терминалов и нетерминалов». Конструкции вроде вложенного `while` и скобок в условии не задаются конечным автоматом без стека, поэтому для них нужен именно **контекстно-свободный** разбор (например, LR).

## Функциональность

Реализован **распознаватель (парсер)** мини-языка с оператором `while`

## Грамматика

```text
<Program>     ::= <Statement>
<Statement>   ::= <While> | <Print> | <Expression> ";"
<While>       ::= "while" "(" <Condition> ")" <Statement>
<Condition>   ::= <Expression> <ComparisonOp> <Expression>
<Expression>  ::= <Primary> | "++" identifier | "--" identifier
<Primary>     ::= <identifier> | <number>
<ComparisonOp> ::= "<" | ">" | "<=" | ">=" | "==" | "!="
<Print>       ::= "print" "(" <Expression> ")" ";"
```

Идентификаторы и числа (целые и вещественные, в т.ч. с порядком `e`/`E`) задаются в лексере.

## Структура каталога

| Файл | Назначение |
|------|------------|
| `lexer.l` | Flex: токены, пробелы, ошибка на недопустимом символе |
| `parser.y` | Bison: правила грамматики, `yyerror` с номером строки |
| `main.c` | Точка входа: открывает файл из аргумента командной строки, задаёт `yyin`, вызывает `yyparse()` |
| `Makefile` | Сборка: `bison -d -o y.tab.c`, `flex`, `gcc` (на macOS выход Bison задаётся явно, чтобы получались `y.tab.c` / `y.tab.h`) |
| `tests/ok/` | Примеры корректных программ (ожидается успешный разбор) |
| `tests/err/` | Примеры с ошибками (ожидается сообщение об ошибке) |

## Сборка и запуск

Требования: **gcc**, **flex**, **GNU bison**.

```bash
cd lab6
make
```

Разбор **только из файла** (путь передаётся первым аргументом):

```bash
./analyzer tests/ok_while_print.txt
```

Автоматическая проверка набора тестов:

```bash
make test
```

Очистка артефактов сборки:

```bash
make clean
```
